# 🏥 한국장기조직기증원(KODA) 웹사이트 리뉴얼 프로젝트

### 한국장기조직기증원(KODA)은 

보건복지부가 지정한 국내 유일의 장기 및 조직 구득기관입니다. 본 프로젝트는 기증자와 유가족을 추모하고 정보를 전달하는 기존 사이트의 사용자 경험(UX)을 현대적으로 개선하기 위해 진행되었습니다.

## 🚀 프로젝트 개요

### 프로젝트 주제
공공기관 웹사이트 UX/UI 리뉴얼 - **정보 위계질서 확립과 사용자 편의를 최우선으로 한 반응형 웹사이트 설계**

### 제작 목적
- **실전 웹 개발 학습**: HTML, CSS, JavaScript를 활용한 실제 공공기관 사이트 구현
- **UX/UI 개선**: 노후화된 디자인을 최신 트렌드에 맞게 세련된 UI로 재구성
- **반응형 설계 마스터**: 요소의 크기 및 굵기 조절을 통해 다양한 기기 환경 최적화
- **정보 구조화**: 복잡한 공공기관의 정보를 한눈에 파악 가능한 위계로 배치

### 사용 기술 스택
- **HTML5**: 시맨틱 마크업을 통한 접근성 고려
- **CSS3**: 반응형 레이아웃 및 애니메이션 구현
- **JavaScript (Vanilla)**: 라이브러리 연동 및 동적 UI 로직 구현
- **Swiper.js**: 메인 배너 및 콘텐츠 슬라이더 라이브러리

## 📄 페이지별 구현 의도 및 설명

### [메인 페이지 (index.html)]

**구현 목표**: 사용자가 서비스를 직관적으로 이해하고 사이트의 기능들을 한눈에 파악 할 수 있도록 구성

**핵심 기능**
- *지능형 GNB(Global Navigation Bar)* : 메인 메뉴 호버 시 각 메뉴에 맞는 서브메뉴가 나타나며, 서브메뉴 영역 이동 시에도 상태가 유지
- *동적 탭 시스템* : 공지사항 및 협력기관 섹션에서 버튼 클릭 시 관련 콘텐츠와 슬라이더가 동시 전환
- *반응형 퀵 메뉴* : 스크롤 위치에 따라 퀵 메뉴의 스타일이 변하여 사용자 편의성 제공

**구현 방법**
- *상태 관리* : let 키워드로 변수를 선언하고, classList.toggle 및 add/remove를 사용하여 UI의 활성 상태를 제어
- *이벤트 위임 및 데이터 속성* : dataset.target이나 getAttribute('data-tab')을 사용하여 클릭한 요소와 노출할 콘텐츠를 1:1로 매칭
- *라이브러리 커스텀* : Swiper.js를 활용하되, autoplay 및 navigation 옵션을 조정하여 프로젝트 성격에 맞는 슬라이더 구현

**어려웠던 점**
- 코드의 중복으로 인한 기능들의 충돌

**개선 방향**
- 현재 탭 메뉴 로직이 섹션별로 유사하게 반복되는데, 이를 하나의 공통 함수로 모듈화하여 재사용성 높이기
- 단순한 클래스 토글 외에 CSS Transition을 보강하여 더 부드러운 메뉴 전환 효과 적용

### [게시판 페이지 (subpage01.html)]

**구현 목표**: 사용자가 서비스를 직관적으로 이해하고 사이트의 기능들을 한눈에 파악 할 수 있도록 구성

**핵심 기능**
- *사용자 정의 페이지네이션* : 서버 없이 클라이언트 사이드에서 대량의 게시글을 일정 개수씩 나누어 출력
- *그룹 페이지 이동* : '이전/다음' 버튼을 통해 페이지 번호 묶음(1~5, 6~10 등)을 이동하는 고차원 로직

**구현 방법**
- *데이터 분할 알고리즘* : Math.ceil로 전체 페이지 수를 계산하고, slice(start, end) 메서드를 사용하여 현재 페이지에 해당하는 데이터만 추출하여 렌더링
- *조건부 렌더링* : displayRow()와 displayPage() 함수를 분리하여, 게시글 목록과 하단 페이지 번호를 독립적이면서도 유기적으로 관리

**어려웠던 점**


**개선 방향**:
- 현재는 하드코딩된 HTML 요소를 기준으로 계산하지만, 향후 JSON 형태의 데이터를 받아와서 동적으로 테이블을 생성하는 로직으로 발전

## 📝 프로젝트 성찰 (Reflection)

### 1. 디자인과 구현 사이의 균형점 탐색
디자인과 프론트엔드 구현을 처음으로 병행하며, **'시각적 이상향'과 '기술적 실현 가능성' 사이의 최적의 접점**을 찾는 데 집중했습니다. 화려한 효과에 치중하기보다 실제 코드로 구현했을 때의 안정성과 퍼포먼스를 고려하며, 디자인과 개발이 유기적으로 연결되는 프로세스를 경험했습니다.

### 2. 정보 위계 설정을 통한 UX 개선
방대한 공공기관의 정보를 사용자가 혼란 없이 수용할 수 있도록 **시각적 위계질서(Visual Hierarchy)**를 확립하는 데 주력했습니다. 요소의 크기, 굵기, 색상 대비를 정밀하게 조정하여 정보의 우선순위를 명확히 함으로써, 리뉴얼의 본질적인 목표인 '사용자 편의성'을 달성하고자 노력했습니다.

### 3. 반응형 웹의 기술적 메커니즘 이해
단순히 화면 크기를 줄이는 것을 넘어, 기기 환경에 따라 폰트와 이미지 배치가 유동적으로 변하는 **반응형 설계의 핵심 원리**를 깊이 있게 학습했습니다. CSS 미디어 쿼리를 활용해 다양한 뷰포트에서도 정보의 가독성이 훼손되지 않도록 정교하게 레이아웃을 조정하는 법을 익혔습니다.

### 4. 한계점 및 향후 과제 (Designer's Perspective)
이번 프로젝트를 통해 **'사용자를 위한 디자인'**과 **'개발자를 위한 디자인'**이라는 두 가지 본질적인 가치를 동시에 실현하고자 노력했습니다. 
향후에는 **사용자에게는 최상의 경험(UX)을 제공하면서도, 개발자에게는 명확한 설계도를 제시할 수 있는 '기술 이해도가 높은 디자이너'**로 거듭나고 싶습니다. 단순히 시각적인 규칙을 전달하는 가이드가 아니라, 실제 구현 시 발생할 수 있는 변수까지 디자인 단계에서 미리 계산하여 '오차 없는 구현'을 가능하게 하는 정교한 디자인 시스템을 구축하는 것이 저의 다음 목표입니다.
